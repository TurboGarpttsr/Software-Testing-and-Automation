# Software-Testing-and-Automation
my repository for my Software Testing and Automation CS-320-16977-M01 class
How can I ensure that my code, program, or software is functional and secure? I test it repeatedly during the coding phase, and after every major part, I test the whole part manually. Using many testing techniques, like unit testing and system testing, closer to the end of the project.
How do I interpret user needs and incorporate them into a program? After collecting a list of what should be in each program or application that I make, I sort it into almost a checklist. If there is a rubric, then I use that as a checklist instead.
How do I approach designing software? I first find out what the program im making needs and turn that into a checklist. After making the checklist i go ahead and start from the ground up and use the checklist as a guide to what the program should become.
